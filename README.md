# Web Novel Automation Tool: Scrape, Translate, and Audiobook/Video Generator

ระบบเครื่องมืออัตโนมัติในการดึงเนื้อหานิยายภาษาจีน แปลเป็นภาษาไทยด้วย Gemini API โดยอ้างอิงและอัปเดตคำศัพท์เฉพาะผ่านพจนานุกรม (`glossary.json`) รวมถึงแปลงเป็นไฟล์เสียงภาษาไทย (TTS) และรวมไฟล์ทั้งหมดออกเป็นไฟล์มัลติมีเดีย `.mp4` พร้อมดึงข้อมูลประทับเวลา (Timestamps) สำหรับใช้งานบน YouTube

---

## 🛠️ ความต้องการของระบบ (Requirements)

1. **Python 3.8 หรือสูงกว่า**
2. **เครื่องมือและไลบรารีที่จำเป็น**:
   * ติดตั้งไลบรารีผ่าน pip:
     ```bash
     pip install -r requirements.txt
     ```
   * ติดตั้งเบราว์เซอร์สำหรับ Playwright Scraper:
     ```bash
     playwright install chromium
     ```

3. **การตั้งค่าไฟล์ `.env` (ที่โฟลเดอร์ Root)**:
   สร้างหรือตรวจสอบไฟล์ `.env` ที่โฟลเดอร์เริ่มต้นของโปรเจกต์ และใส่คีย์ API ลงไป (หากใช้ DeepSeek ให้ระบุทั้งสองคีย์หรือเฉพาะคีย์ที่ใช้งาน):
   ```env
   GEMINI_API_KEY=your_gemini_api_key_here
   DEEPSEEK_API_KEY=your_deepseek_api_key_here
   ```

## 📁 โครงสร้างโปรเจกต์และการทำงานของไฟล์

* `main.py` - จุดเริ่มต้นการเรียกใช้คำสั่งควบคุมการทำงานทั้งหมด (CLI)
* `scraper.py` - ดึงข้อมูลหน้าเว็บนิยายจีนแบบทีละหน้าตามลำดับหน้าถัดไป
* `translator.py` - แปลเป็นไทยด้วย Gemini API และอ้างอิงตารางคำแปล `glossary.json`
* `audiobook.py` - สร้างไฟล์เสียง MP3 (ใช้ `edge-tts`) และประกอบเข้ากับภาพหน้าปกเป็นวิดีโอ `.mp4` (ใช้ `static-ffmpeg`)
* `utils.py` - เครื่องมือจัดการเกี่ยวกับข้อความ, บาร์แสดงสถานะ (Progress Bar) และชื่อไฟล์
* `glossary.json` - ตารางคู่คำศัพท์ จีน -> ไทย (ระบบจะค้นคำศัพท์ที่ตรงกันในเนื้อหาและส่งไปเป็น Reference คำแปลให้ AI เสมอ)

## 🚀 วิธีการสั่งรันโปรแกรม (How to Run)
 
> [!TIP]
> ปัจจุบันโปรแกรมรองรับตัวเลือก `--option` (ระบุที่เก็บไฟล์งาน) ทำให้คุณไม่จำเป็นต้องสลับโฟลเดอร์ใน Terminal โดยตรง เพียงระบุโฟลเดอร์ปลายทางที่ต้องการเก็บไฟล์งาน ระบบจะคอยสลับและคุมโฟลเดอร์ให้เอง

### โครงสร้างโฟลเดอร์ภายในที่กำหนดโดย `--option`
เมื่อรันคำสั่งโดยใช้การระบุที่เซฟงานด้วย `--option "/path/to/folder"` ระบบจะสร้างและใช้โครงสร้างดังนี้:
* `[โฟลเดอร์หลัก]` - ใช้เซฟไฟล์ดิบภาษาจีนจากการดึงหน้าเว็บ (Scrape)
* `[โฟลเดอร์หลัก]/done/` - ใช้ย้ายเก็บไฟล์ดิบภาษาจีนที่ผ่านการแปลเรียบร้อยแล้ว
* `[โฟลเดอร์หลัก]/translate/` - ใช้เซฟเก็บไฟล์แปลภาษาไทย (.md)
* `[โฟลเดอร์หลัก]/Audiobook/` - โฟลเดอร์รวมงานเสียง ประกอบด้วย:
  * ไฟล์เสียงรายตอน `.mp3`
  * ไฟล์บอกจุดเวลาเริ่มต้นของแต่ละตอน: `timetrack.txt` (สำหรับ Chapters บน YouTube)
  * ไฟล์รวมบทเสียงยาว: `combined_audiobook.mp3`
  * ไฟล์วิดีโอนิ่งรวมสมบูรณ์เปิดเล่นได้ทันที: `combined_audiobook.mp4`

### วิธีการรันแบบรวมทุกขั้นตอนรวดเดียว (Stage 1-3)
คำสั่งนี้จะทำการดึงข้อมูล แปลภาษา และสร้างวิดีโอจัดรวมในคำสั่งเดียว:
```bash
python3 main.py all --url "URL_เริ่มต้น" --limit จำนวนตอน --title-selector "CSS_Title" --content-selector "CSS_Content" --next-selector "CSS_Next_Link" --option "โฟลเดอร์ที่เก็บงาน"
```
* **ตัวเลือกการจำกัดจำนวนตอน (`--limit`)**: สามารถใส่จำนวนตอนที่ต้องการ หรือป้อน `null` เพื่อสั่งให้โปรแกรมดึงตอนใหม่เรื่อย ๆ ไปจนสุดทางโดยอัตโนมัติ (เช่น `--limit null`)

*ตัวอย่างการรันจริงสำหรับเรื่องจิงอันโหว:*
```bash
python3 main.py all --url "https://funs.me/text/17561/15670001.html" --limit 5 --title-selector "td[background*='bgheader']" --content-selector "#ChSize" --next-selector "a.pages" --option "/Users/chettatosuanchit/Downloads/จิงอันโหว"
```

*ตัวอย่างการรันจริงสำหรับเรื่องคนบ้าแห่งต้าหมิง:*
```bash
python3 main.py all --url "https://www.bgg99.cc/book/1136853033/916083008.html" --limit 25 --title-selector ".content h1" --content-selector "#content" --next-selector ".page_chapter ul li:nth-child(3) a" --option "/Users/chettatosuanchit/Documents/คนบ้าแห่งต้าหมิง" --ai "deepseek"
```


*ตัวอย่างการรันจริงสำหรับเรื่องช่วงเวลาหลายปีของฉันในฐานะนักบวชลัทธิเต๋า:*
```bash
python3 main.py all --url "https://funs.me/text/2080/15540101.html" --limit 25 --title-selector "td[background*='bgheader'] > font" --content-selector "#ChSize" --next-selector "a.pages" --option "/Users/chettatosuanchit/Documents/ช่วงเวลาหลายปีของฉันในฐานะนักบวชลัทธิเต๋า" --ai "deepseek"
```

### วิธีการสั่งรันแยกทีละขั้นตอน (Stage-by-Stage)

#### **ขั้นตอนที่ 1: ดึงข้อมูลต้นฉบับภาษาจีน (Scrape)**
ดึงนิยายตอนภาษาจีนมาเก็บไว้ในเครื่องเป็นไฟล์ `.md` เซฟลงโฟลเดอร์งานที่เลือก:
```bash
python3 main.py scrape --url "URL_เริ่มต้น" --limit จำนวนตอน --title-selector "CSS_Title" --content-selector "CSS_Content" --next-selector "CSS_Next_Link" --option "โฟลเดอร์ที่เก็บงาน"
```
* **ตัวเลือกเสริม**:
  * `--limit`: จำนวนตอนที่ต้องการดึง (ค่าเริ่มต้น: 5) หรือกำหนดเป็น `null` เพื่อดึงข้อมูลตอนต่อไปเรื่อย ๆ แบบไม่จำกัด
  * `--proxy`: ใช้กำหนดค่า Proxy สำหรับ Scraper (เช่น ใส่ `--proxy "true"` เพื่อเรียกใช้พร็อกซีองค์กร `http://siph-mmswg01.siph.com:8080` หรือระบุพร็อกซีแบบกำหนดเอง เช่น `--proxy "http://host:port"`)




#### **ขั้นตอนที่ 2: แปลเนื้อหาเปรียบเทียบคำศัพท์ (Translate)**
อ่านไฟล์ดิบภาษาจีนในโฟลเดอร์งานที่เลือก และทำการแปลภาษาด้วยระบบ AI:
```bash
python3 main.py translate --option "โฟลเดอร์ที่เก็บงาน"
```
* **ตัวเลือกเสริม**:
  * `--ai`: เลือกผู้ให้บริการแปลภาษาระหว่าง `gemini` หรือ `deepseek` (สามารถใช้พิมพ์เว้นวรรคหรือขีดกลางได้ เช่น `deep seek`, `deep-seek` จะถูกแปลงเป็น `deepseek` อัตโนมัติ โดยค่าเริ่มต้นคือ `gemini`)
  * `--model`: ระบุชื่อโมเดลเฉพาะเจาะจง (ค่าเริ่มต้น: `gemini-2.5-flash` สำหรับ Gemini, `deepseek-v4-flash` สำหรับ DeepSeek)
  * `--limit`: กำหนดจำนวนไฟล์ที่จะแปลในรอบนี้ (เช่น ใส่ `--limit 10` เพื่อแปลแค่ 10 ไฟล์ล่าสุดที่ดึงมา เหมาะกับการทำทีละเซ็ต)
* **กลไกอัจฉริยะเพิ่มเติม**:
  * **DeepSeek Batching & Caching**: เมื่อใช้ `deepseek` ระบบจะทำการนับและแบ่งกลุ่มตอนการแปลทีละ 10 ตอน (เศษเหลือต่ำกว่า 5 จะถูกปัดไปรวมกับก้อนก่อนหน้า เช่น 44 ตอน จะได้เป็นกลุ่ม 10, 10, 10, 14) จากนั้นค้นหาคำศัพท์ที่ตรงกันใน `glossary.json` เพื่อทำแคช `snapshot_glossary.json` สำหรับใช้ทำ Prompt Cache Hit ในการแปลช่วงนั้น และจะทำการลบไฟล์ snapshot ทิ้งให้หลังจากจบแต่ละก้อน
  * เมื่อแปลสำเร็จ: ไฟล์ดิบจีนจะย้ายไปที่ `done/` และไฟล์แปลไทยจะย้ายไปที่ `translate/` โดยอ้างอิงและอัปเดตศัพท์คู่แปลเข้าสู่ `glossary.json` ตลอดเวลา

#### **ขั้นตอนที่ 3: สร้างเสียงและวิดีโอ (Audiobook compilation)**
แปลงข้อความภาษาไทยที่แปลแล้วในโฟลเดอร์ `translate/` ให้เป็นไฟล์เสียงทีละตอน และสร้างไฟล์วิดีโอรวม:
```bash
python3 main.py audiobook --combine --option "โฟลเดอร์ที่เก็บงาน"
```
* **ระบบเช็คข้ามการสร้างไฟล์เสียง (Skip Logic)**: ตอนสร้าง MP3 ระบบจะเช็คก่อนว่ามีไฟล์เสียงของตอนนั้น ๆ รันเสร็จและอยู่ในโฟลเดอร์ `Audiobook/` หรือยัง ถ้ามีแล้วระบบจะ **Skip** ข้ามไปทันทีเพื่อความรวดเร็วและไม่สร้างเสียงซ้ำ
* **ตัวเลือกเสริม**:
  * `--voice`: ใช้กำหนดรหัสเสียงของ Edge-TTS (ค่าเริ่มต้น: `th-TH-NiwatNeural` สำหรับเสียงผู้ชาย)
  * `--no-video`: สำหรับต้องการให้สร้างเฉพาะไฟล์เสียง MP3 ไม่ประกอบวิดีโอรวม `.mp4`
  * `--limit`: กำหนดจำนวนไฟล์แปลที่ต้องการแปลงเป็นไฟล์เสียง/วิดีโอในรอบนี้ (เช่น ใส่ `--limit 5` เพื่อทยอยประมวลผลทีละ 5 ตอน)
* หลังจากประมวลผลสำเร็จ: ไฟล์รวมเสียง ไฟล์บอกจุดเวลา และไฟล์วิดีโอนิ่งรวมเสียง จะไปอยู่ใต้โฟลเดอร์ `Audiobook/`

## 🔍 วิธีการหาและใช้ CSS Selectors สำหรับผู้ใช้งาน

สำหรับผู้ใช้งานที่ต้องการหาค่า CSS Selectors เพื่อป้อนให้กับตัวเลือก `--title-selector`, `--content-selector` และ `--next-selector` สามารถใช้วิธีง่าย ๆ ด้านล่างนี้เพื่อค้นหาผ่านหน้าเว็บเบราว์เซอร์ (เช่น Google Chrome, Safari หรือ Firefox) ได้เจ้าค่ะ:

### 1. วิธีเปิดเครื่องมือพัฒนา (Inspect Element)
1. เปิดเบราว์เซอร์ไปที่หน้าเว็บนิยายตอนที่ต้องการดึงข้อมูล
2. คลิกขวาตรงจุดที่ต้องการหา (เช่น บนชื่อตอน หรือตรงเนื้อหาของนิยาย)
3. เลือก **Inspect (ตรวจสอบ)** หรือกดคีย์ลัด `F12` (สำหรับ Windows) หรือ `Cmd + Option + I` (สำหรับ Mac)
4. จะมีหน้าต่างเครื่องมือผู้พัฒนา (Developer Tools) ปรากฏขึ้นมาที่ด้านข้างหรือด้านล่างของจอภาพ

### 2. วิธีค้นหา CSS Selector ของแต่ละส่วน

#### **A. ตัวดึงชื่อตอน (`--title-selector`)**
* **เป้าหมาย**: ชื่อเรื่องหรือชื่อตอนนิยาย (เช่น *第215章 都是你吃的*)
* **การหา**: คลิกขวาที่ชื่อตอนบนหน้าเว็บแล้วกด Inspect เพื่อดูแท็ก HTML ของชื่อตอน
* **รูปแบบทั่วไป**: 
  * มักอยู่ในแท็ก `<h1>`, `<h2>` หรือแท็กที่มี class เช่น `h1.title` หรือ `.chapter-title`
  * **ตัวอย่างการป้อนค่า**:
    * หากเป็นแท็ก `<h1>` ธรรมดา ใส่เป็น: `"h1"`
    * หากเป็นแท็กที่มี class เช่น `<h1 class="chapter-title">` ใส่เป็น: `".chapter-title"` หรือ `"h1.chapter-title"`
    * หากเป็นตารางเฉพาะ เช่น `<td class="bgheader">` ใส่เป็น: `"td.bgheader"` หรือ `"td[background*='bgheader']"`

#### **B. ตัวดึงเนื้อหา (`--content-selector`)**
* **เป้าหมาย**: กล่องข้อความขนาดใหญ่ที่คลุมเนื้อความนิยายทั้งหมดของตอนนั้น
* **การหา**: คลิกขวาตรงเนื้อความสักบรรทัดนึง แล้วกด Inspect จากนั้นเลื่อนแถบโค้ดขึ้นไปทีละชั้นเพื่อมองหาแท็กครอบนอก (Parent `<div>` หรือ `<section>`) ที่เวลาเอาเมาส์ชี้แล้วจะไฮไลท์พื้นที่เนื้อหาทั้งหมดของนิยาย
* **รูปแบบทั่วไป**:
  * มักเป็นกล่องที่มี ID หรือ Class พิเศษ
  * **ตัวอย่างการป้อนค่า**:
    * หากโค้ดเป็น `<div id="content">` ใส่เป็น: `"#content"` (เครื่องหมาย `#` แทน ID)
    * หากโค้ดเป็น `<div id="ChSize">` ใส่เป็น: `"#ChSize"`
    * หากโค้ดเป็น `<div class="read-content">` ใส่เป็น: `".read-content"` (เครื่องหมาย `.` แทน Class)
    * หากโค้ดเป็น `<article>` ใส่เป็น: `"article"`

#### **C. ตัวคลิกหน้าถัดไป (`--next-selector`)**
* **เป้าหมาย**: ปุ่ม ลิงก์ หรือตัวอักษรที่กดเพื่อเลื่อนไปตอนถัดไป (เช่น *下一章*, *Next*, *ตอนถัดไป*)
* **การหา**: คลิกขวาที่ปุ่ม "ตอนถัดไป" แล้วกด Inspect เพื่อดูแท็กลิงก์ `<a>` ที่ใช้ข้ามหน้า
* **รูปแบบทั่วไป**:
  * มักเป็นแท็กลิงก์ที่มี class หรือข้อความเฉพาะ เช่น `<a href="..." class="next">下一章</a>`
  * **ตัวอย่างการป้อนค่า**:
    * หากโค้ดเป็น `<a class="next">` ใส่เป็น: `"a.next"` หรือ `".next"`
    * หากเป็นลิงก์ที่มีการระบุคลาสแบบแบ่งหน้าหลาย ๆ ปุ่ม เช่น `<a class="pages">` ใส่เป็น: `"a.pages"`
    * หากเป็นปุ่มไอดีเฉพาะ เช่น `<a id="next_chapter">` ใส่เป็น: `"a#next_chapter"` หรือ `"#next_chapter"`

### 💡 เคล็ดลับทางลัดในการคัดลอก (Copy Selector)
หากไม่มั่นใจว่าจะเขียน Selector เองอย่างไร ให้ใช้ฟังก์ชันช่วยของเบราว์เซอร์ได้เจ้าค่ะ:
1. ในหน้าต่าง Inspect คลิกขวาที่แถวโค้ดของส่วนที่ต้องการ (เช่น บรรทัด `<div id="content">`)
2. เลือกเมนู **Copy** ➡️ **Copy selector**
3. เบราว์เซอร์จะคัดลอกค่าสำหรับนำมาใช้มาให้ทันที (อาจได้ค่ามาเป็น `body > div.main > div#content` ซึ่งสามารถนำมาใส่ในคำสั่งตัวเลือกได้ทันทีเจ้าค่ะ)

> [!WARNING]
> สำหรับค่า CSS Selector ที่มีเครื่องหมายคำพูดเดี่ยวหรือข้อความพิเศษ เช่น `td[background*='bgheader']` เมื่อรันคำสั่งใน Terminal ให้ทำการ **ครอบค่าตัวเลือกทั้งหมดด้วยเครื่องหมายคำพูด (Quotes)** เสมอ เพื่อป้องกันไม่ให้ระบบ Shell สับสน ตัวอย่างเช่น:
> ```bash
> --title-selector "td[background*='bgheader']"
> ```

## 📝 ข้อกำหนดพิเศษเพิ่มเติม

* **แถบแสดงสถานะ (Progress Bar)**: ระบบมีแถบแสดงความคืบหน้าแบบอักษร `[██████░░░░] 60% (3/5)` ช่วยให้เห็นความคืบหน้าการทำงานตลอดเวลาในทุกส่วนงาน
* **การป้องกันความเข้ากันได้ของเสียงและวิดีโอ (QuickTime & macOS Compatibility)**: ระบบวิดีโอใช้ฟิลเตอร์จัดการภาพนิ่งประสิทธิภาพสูงที่ถอดรหัสรูปภาพเพียงครั้งเดียวพร้อมการเรนเดอร์เสียงในระดับมาตรฐานสากล `44100Hz Stereo AAC` และเสริมออปชัน `-movflags +faststart` เพื่อเลื่อนข้อมูลดัชนีภาพและเสียงไปไว้จุดเริ่มต้น ทำให้เปิดไฟล์รวมขนาดใหญ่ได้อย่างรวดเร็วและปลอดภัยบน QuickTime Player ของ macOS รวมถึงเปิดบนโทรศัพท์มือถือทุกรุ่นได้อย่างราบรื่น
