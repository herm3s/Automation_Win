# Web Novel Automation Tool: Scrape, Translate, and Audiobook/Video Generator

ระบบเครื่องมืออัตโนมัติในการดึงเนื้อหานิยายภาษาจีน แปลเป็นภาษาไทยด้วย Gemini API โดยอ้างอิงและอัปเดตคำศัพท์เฉพาะผ่านพจนานุกรม (`glossary.json`) รวมถึงแปลงเป็นไฟล์เสียงภาษาไทย (TTS) และรวมไฟล์ทั้งหมดออกเป็นไฟล์มัลติมีเดีย `.mp4` พร้อมดึงข้อมูลประทับเวลา (Timestamps) สำหรับใช้งานบน YouTube

---

## 🛠️ ความต้องการของระบบ (Requirements)

1. **Python 3.8 หรือสูงกว่า**
2. **เครื่องมือและไลบรารีที่จำเป็น**:
   * ติดตั้งไลบรารีผ่าน pip:
     ```bash
     pip install -r requirements.txt
     ```
   * ติดตั้งเบราว์เซอร์สำหรับ Playwright Scraper:
     ```bash
     playwright install chromium
     ```

3. **การตั้งค่าไฟล์ `.env` (ที่โฟลเดอร์ Root)**:
   สร้างหรือตรวจสอบไฟล์ `.env` ที่โฟลเดอร์เริ่มต้นของโปรเจกต์ และใส่คีย์ API ลงไป (หากใช้ DeepSeek ให้ระบุทั้งสองคีย์หรือเฉพาะคีย์ที่ใช้งาน):
   ```env
   GEMINI_API_KEY=your_gemini_api_key_here
   DEEPSEEK_API_KEY=your_deepseek_api_key_here
   ```

## 📁 โครงสร้างโปรเจกต์และการทำงานของไฟล์

* `main.py` - จุดเริ่มต้นการเรียกใช้คำสั่งควบคุมการทำงานทั้งหมด (CLI)
* `scraper.py` - ดึงข้อมูลหน้าเว็บนิยายจีนแบบทีละหน้าตามลำดับหน้าถัดไป
* `translator.py` - แปลเป็นไทยด้วย Gemini API และอ้างอิงตารางคำแปล `glossary.json`
* `audiobook.py` - สร้างไฟล์เสียง MP3 (ใช้ `edge-tts`) และประกอบเข้ากับภาพหน้าปกเป็นวิดีโอ `.mp4` (ใช้ `static-ffmpeg`)
* `utils.py` - เครื่องมือจัดการเกี่ยวกับข้อความ, บาร์แสดงสถานะ (Progress Bar) และชื่อไฟล์
* `glossary.json` - ตารางคู่คำศัพท์ จีน -> ไทย (ระบบจะค้นคำศัพท์ที่ตรงกันในเนื้อหาและส่งไปเป็น Reference คำแปลให้ AI เสมอ)

## 🚀 วิธีการสั่งรันโปรแกรม (How to Run)

> [!TIP]
> ปัจจุบันโปรแกรมรองรับตัวเลือก `--option` หรือ `--output` (ระบุที่เก็บไฟล์งาน) ทำให้คุณไม่จำเป็นต้องสลับโฟลเดอร์หรือใช้ `../main.py` ใน Terminal โดยตรง เพียงระบุโฟลเดอร์ปลายทางที่ต้องการเก็บไฟล์งาน (เช่น `D:\Gemini Labs\จิงอันโหว`) ระบบจะคอยสลับและคุมโฟลเดอร์ให้เอง

### โครงสร้างโฟลเดอร์ภายในที่กำหนดโดย `--option` (หรือ `--output`)
เมื่อรันคำสั่งโดยระบุที่เซฟงานด้วย `--option` หรือ `--output` ระบบจะสร้างและใช้โครงสร้างดังนี้:
* `[โฟลเดอร์หลัก]` - ใช้เซฟไฟล์ดิบภาษาจีนจากการดึงหน้าเว็บ (Scrape)
* `[โฟลเดอร์หลัก]/done/` - ใช้ย้ายเก็บไฟล์ดิบภาษาจีนที่ผ่านการแปลเรียบร้อยแล้ว
* `[โฟลเดอร์หลัก]/translate/` - ใช้เซฟเก็บไฟล์แปลภาษาไทย (.md)
* `[โฟลเดอร์หลัก]/Audiobook/` - โฟลเดอร์รวมงานเสียง ประกอบด้วย:
  * ไฟล์เสียงรายตอน `.mp3`
  * ไฟล์บอกจุดเวลาเริ่มต้นของแต่ละตอน: `timetrack.txt` (สำหรับ Chapters บน YouTube)
  * ไฟล์รวมบทเสียงยาว: `combined_audiobook.mp3`
  * ไฟล์วิดีโอนิ่งรวมสมบูรณ์เปิดเล่นได้ทันที: `combined_audiobook.mp4`

### วิธีการสั่งรันแยกทีละขั้นตอน (Stage-by-Stage)

#### **ขั้นตอนที่ 1: ดึงข้อมูลต้นฉบับภาษาจีน (Scrape)**
ดึงนิยายตอนภาษาจีนมาเก็บไว้ในเครื่องเป็นไฟล์ `.md` เซฟลงโฟลเดอร์งานที่เลือก:
```bash
python main.py scrape --url "URL_เริ่มต้น" --limit จำนวนตอน --title-selector "CSS_Title" --content-selector "CSS_Content" --next-selector "CSS_Next_Link" --option "โฟลเดอร์ที่เก็บงาน"
```

#### **ขั้นตอนที่ 2: แปลเนื้อหาเปรียบเทียบคำศัพท์ (Translate) ด้วยกลยุทธ์ Two-Pass Strategy**
การแปลภาษาจีนเป็นภาษาไทยจะทำงานด้วยระบบ **Two-Pass Strategy** เพื่อคุณภาพและความสม่ำเสมอของคำศัพท์ และเพื่อประโยชน์ในการทำ Caching:
* **Pass 1: Glossary Extraction (สกัดคำศัพท์)**:
  - ระบบจะสแกนไฟล์ `.md` ต้นฉบับทั้งหมด และแบ่งกลุ่ม (Batch ละ 10 ไฟล์) เพื่อให้ AI (แนะนำ `gemini-2.5-flash` ที่ราคาประหยัดและรวดเร็ว) สกัดเฉพาะคำเฉพาะ เช่น ชื่อคน, สถานที่, นิกาย, ยศขุนนาง และสิ่งของ นำมาจัดเก็บรวมไว้ในพจนานุกรมที่สมบูรณ์ `complete_glossary.json`
  - มีระบบบันทึกความคืบหน้าไว้ใน `extraction_state.json` ทุกๆ 10 ไฟล์ ทำให้สามารถกู้คืนหรือรันต่อ (Resume) ได้หากโปรแกรมหยุดชะงักระหว่างดึงคำศัพท์
* **Pass 2: Unified Translation (แปลจริงผ่าน System Prompt Caching)**:
  - ระบบจะดึงคำศัพท์ที่จับคู่ตรงกับบทที่จะแปลทั้งหมด (สูงสุด 500 คำ) ไปฝังไว้ใน **System Prompt** ทำให้ System Prompt ของทุกตอนคงที่เหมือนกัน 100% ตลอด Pass 2
  - การทำเช่นนี้ทำให้การใช้งานร่วมกับ DeepSeek เกิด **Prefix Cache Hit สูงกว่า 80%-100%** ช่วยลดจำนวนการใช้ Token และประหยัดค่าใช้จ่ายอย่างมาก
  - ระหว่างแปลจริงใน Pass 2 จะไม่มีการส่งคำศัพท์กลับมาแก้/เพิ่มเข้า `complete_glossary.json` เพื่อป้องกันไม่ให้โครงสร้าง System Prompt เปลี่ยนแปลงจนหลุด Cache
  - แปลเสร็จแล้วระบบจะแสดงผลลัพธ์อัตรา **Cache Hit Rate** ของ DeepSeek รายตอน และจะคำนวณสรุปอัตรา Cache Hit เฉลี่ยรวมให้ที่ปลายทาง

```bash
python main.py translate --option "โฟลเดอร์ที่เก็บงาน"
```
* **ตัวเลือกเสริม**:
  * `--ai`: เลือกผู้ให้บริการแปลภาษา ระหว่าง `gemini` หรือ `deepseek` (ค่าเริ่มต้น: `gemini`)
  * `--model`: ระบุชื่อโมเดลเฉพาะเจาะจงที่ต้องการ (ค่าเริ่มต้น: `gemini-2.5-flash` สำหรับ Gemini, `deepseek-v4-flash` สำหรับ DeepSeek)
* หลังจากแปลสำเร็จ: 
  * ไฟล์แปลภาษาไทยจะถูกสร้างไว้ในโฟลเดอร์ `translate/` ทันที
  * ไฟล์ดิบภาษาจีนต้นฉบับจะถูกย้ายเข้าโฟลเดอร์ `done/` อัตโนมัติ เพื่อป้องกันการสั่งแปลซ้ำ
  * ไฟล์ระบุสถานะชั่วคราวอย่าง `extraction_state.json` จะถูกลบออกโดยอัตโนมัติเมื่อสิ้นสุด Pass 2 อย่างสมบูรณ์

#### **ขั้นตอนที่ 3: สร้างเสียงและวิดีโอ (Audiobook compilation)**
แปลงข้อความภาษาไทยที่แปลแล้วให้เป็นไฟล์เสียงทีละตอน รวมเสียงทั้งหมด และสร้างไฟล์วิดีโอ `.mp4` หน้าปกนิ่งสำหรับการอัปโหลดลงสื่อมัลติมีเดีย:
```bash
python main.py audiobook --combine --option "โฟลเดอร์ที่เก็บงาน"
```
* **ตัวเลือกเสริม**:
  * `--voice`: ใช้กำหนดรหัสเสียงของ Edge-TTS (ค่าเริ่มต้น: `th-TH-NiwatNeural` สำหรับเสียงผู้ชาย)
  * `--no-video`: สำหรับต้องการให้สร้างเฉพาะไฟล์เสียง MP3 ไม่ประกอบวิดีโอ `.mp4`
* หลังจากประมวลผลสำเร็จ ไฟล์ทั้งหมดจะถูกบันทึกไว้ในโฟลเดอร์ **`Audiobook/`**:
  * `000X_*.mp3`: ไฟล์เสียงของแต่ละตอน
  * `combined_audiobook.mp3`: ไฟล์รวมบทเสียงยาวทั้งหมด
  * `combined_audiobook.mp4`: ไฟล์วิดีโอนิ่งความละเอียดประหยัดเนื้อหา
  * `timetrack.txt`: ไฟล์บอกจุดเวลาเริ่มต้นของแต่ละตอน (สำหรับใส่ใต้คลิป YouTube เป็น Chapters)
